Individual Developer Workflow

Follow this daily routine to stay organized and avoid conflicts.

1. Setup Git (One Time)

git config --global user.name "Your Name"
git config --global user.email "you@email.com"

2. Start Your Day

Step 1: Get latest code from develop branch 
git checkout develop
git pull origin develop
Step 2: Create your feature branch 
git checkout -b feature/login-page

3. Work on Your Code

Tip: Work only in your feature branch. Never directly in develop or main.

4. Commit Your Changes

Step 1: Stage your changes 
git add .  # Add all files
# OR
git add src/components/Login.jsx  # Add specific file
Step 2: Commit with clear message 
git commit -m "Add login form UI with validation"

5. Push to GitHub

git push origin feature/login-page

6. Handle Conflicts (If They Occur)

Warning: Conflicts happen when two people edit the same file. Don't panic! 
git checkout feature/my-feature
git pull origin develop  # Get latest changes

# Resolve conflicts manually (Git marks them with <<<<<<< and >>>>>>>)
git add conflicted-file.js
git commit -m "Resolve merge conflict"
git push origin feature/my-feature

Team Collaboration Workflow

How to submit, review, and merge code with your team.

1. Create Pull Request (PR)

Step 1: Go to GitHub → your repo → "Compare & pull request"
Step 2: Set base: develop ← compare: feature/login-page
Step 3: Write description of what you did
Step 4: Assign team leader as reviewer

2. Code Review Process

Reviewer Checklist:
  • Does the code work as described?
  • Are there any obvious bugs?
  • Is the code clean and readable?
  • Are there tests if required?

3. Merge to Develop

Step 1: Reviewer approves PR
Step 2: Click "Merge pull request"
Step 3: Delete your feature branch 
git checkout develop
git pull origin develop
git branch -d feature/login-page

4. Release to Main

Step 1: Create release branch (Leader only) 
git checkout develop
git pull origin develop
git checkout -b release/v1.0
Step 2: After testing, merge to main 
git checkout main
git merge release/v1.0
git tag -a v1.0 -m "Version 1.0 release"
git push origin main --tags

Quick Commands Reference

# Essential Commands
git status          # See what changed
git log --oneline   # View commit history
git diff            # See exact line changes
git stash           # Save unfinished work
git stash pop       # Restore stashed work
git branch          # List all branches
git branch -d branch-name  # Delete merged branch

# Branch Management
git checkout -b new-branch    # Create and switch to new branch
git checkout develop        # Switch to develop
git pull origin develop     # Get latest changes

Team Rules (MUST FOLLOW)

  • Always pull before you push — prevents conflicts
  • Never commit directly to main or develop — everything goes through PRs
  • One feature = one branch — keeps work organized
  • Write meaningful commit messages — future you will thank you
  • Delete your branch after merging — keeps repo clean
  • Use .gitignore — don't commit node_modules, .env, etc.

Troubleshooting Common Issues

Issue: "Merge conflict"

# Resolve conflicts
git checkout feature/my-feature
git pull origin develop

# Edit files to resolve conflicts (look for <<<<<<< and >>>>>>>)
git add .
git commit -m "Fix merge conflicts"
git push origin feature/my-feature

Issue: "Branch already exists"

# Delete local branch first
git branch -d feature/name
# Then create again
git checkout -b feature/name

Issue: "Permission denied"

Fix: Ask team leader to add you as collaborator on GitHub repo.

Issue: "Not on any branch"

# Get back to develop
git checkout develop
git pull origin develop
